一.索引的优缺点:
  优点
     1.所有的mysql列类型(字段类型)都可以被索引,也就是可以给任意字段设置索引
     2.大大加快数据的查询速度.
  缺点:
     1.创建索引和维护要耗费时间,并且随着数据的增加所耗费的时间也会增加.
     2.索引需要占空间,数据表中的数据也会有最大上限设置的,如果有大量的索引,索引文件可能会比数据文件更快达到上限值
     3.对表中的数据进行增,删,改时,索引也需要动态维护.降低了数据的维护速度.
  所以:通过上面所说,我们应该可以知道,并不是每个字段设置索引就好,也不是索引越多越好.而是需要自己的合理使用.


二.索引使用原则:
  1.经常更新的表应避免添加索引,经常查询的字段应该添加索引.
  2.数据量小的就不需要添加索引,由于数据较少,可能查询全部数据的时间比遍历索引的时间还要少.(一般数据几十万以上才添加)
  3.在一个字段上不同值较少的不要添加索引,比如性别,只有男和女,同一个字段不同值较多的可建立索引.
  总结:经常查询,数据多,复杂.

三.索引分类:
  1.普通索引:Mysql中基本索引类型,没有什么限制,允许在定义索引的列中插入重复值和空值,纯粹是为了提高查询速度.
  2.唯一索引:索引中的值必须是唯一的,但是允许是空值.
  3.主键索引:是一种特殊的索引,不允许有空值.
  4.组合索引:在表中多个字段组合时创建索引,只有使用了组合索引的最左边字段时,才会使用这种索引.叫做最左前缀集合.
             例:create index(id,name,sex)  只查询name和sex不会使用索引.


四.使用索引:
   1.创建普通索引
     create index 索引的名字 on 表名(字段名);
                   ind_name     table(column)
   2.创建唯一索引:
     create unique index 索引的名字 on 表名(字段名)
   
   3.创建复合索引:
     create index ind_name on table(column1,column2)
     也可以创建符合唯一索引,加上unique关键字就好了.
     
   
   3.删除索引:(哪种索引都直接删除就好了)
     drop index ind_name on table;




小建议:在一张数据表中,索引最好不要超过三个(除了主键以外).
