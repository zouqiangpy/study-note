一.安装git:
 下载git,直接傻瓜式安装,一直下一步即可.
  
 注意:其中有个选项必须要勾选,checkout Windom-style,commit Unix-style line endings.就是他会给你默认转换换行符,因为不同操作系统化的换行符是不一样的.别人从linux系统提交      的,你从window系统拉取下来,换行符就会不一样
 检查是否安装完成:
      直接鼠标右键,数来GUI和Bash即可.


二.了解Git.
   1.结构.
     a.工作区(就是我们代码直接保存的地方),通过git add保存代码到暂存区.
     b.暂存区(临时存储,里面的数据即可以commit到本地库中,也可以回退到工作区中).通过git commit保存代码到本地库.
     c.本地库(存储我们历史版本的地方)
三.Git和代码托管中心(github) 
     代码托管中心的主要任务是维护远程仓库.
   1.代码托管中心的分类.
     a.局域网环境下:
        gitlab:比如在一个公司内网中想要搞一个本公司的远程代码托管中心.
     b.外网环境下:
        Github
        gitee(码云)
        coding.net

四.本地库和远程库的交互方式.
   1.团队内部协作.
     a.git push:将本地库的中的版本推送到远程库中.
     b.git clone:复制远程库中的版本(本地库中的版本是一个从无到有的过程)
     c.git pull:跟新远程库和本地库中版本中不一样的地方.
   2.跨团队协作:
     a.就是需要另搞一个远程库b,Fock(跟踪)原来的远程库a.
     b.对远程库b中进行更新,之后需要pull Request远程库a.
     c.远程库a中的人审核同意后,就可以merge(合并).
